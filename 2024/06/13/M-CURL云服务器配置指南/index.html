Conda环境配置

Create mcurl Environment

首先创造mcurl项目的独有环境，与其他环境区分开互不干扰。我们通过.yml的配置文件进行初始配置

然后conda会自动下载yml文件中的设置，然而多数情况下，yml中的pip install部分会失败。但是pytorch等将成功安装。注意此时的pytorch版本应该是新版本，将在之后设置cuba时进行参考。

Install mujoco

可以参考https://zhuanlan.zhihu.com/p/352304615

mujoco是一个物理模拟平台，mcurl基于此运行。

首先在本地电脑前往官网https://www.roboti.us/index.html下载mujoco软件包以及licence key https://www.roboti.us/license.html。压缩后打包上传至服务器的对应网盘。

然后通过终端进入云服务器系统，输入ls可以看到系统的文件结构。

之后在home目录下，创建隐藏文件夹/mujoco，然后将自己的mjkey.txt拷贝到.mujoco文件夹中

注意：必须把/.mujoco/mujoco200_linux修改为~/.mujoco/mujoco200,否则import mujoco_py的时候会报错

最终mjkey.txt存放的路径为~/.mujoco/mujoco200/bin下，注意解压缩完的路径可能有两层mujoco200文件夹，去掉最外层即可。（未遇到此现象）

运行与修改python代码

为了方便起见，我们直接放弃通过sh脚本运行python，而是直接运行train.py来实现。然而运行train.py之前，可能会有划黄线的库，那么就把他安装好，典型的有 color 库以及 sci-image库， 网上都能找到安装命令，一般不会出问题。

在运行之前，更改部分argument的default参数就可以调参。注意可以把save_video调成false可以少de一些bug.

然后直接在终端中运行train.py（能跑起来就有鬼了），会遇见各种小问题，直接去对应的库文件中按照报错修改。典型的有

那就把gym.envs.registry.env_specs后面的env_specs删掉就好了。https://github.com/openai/gym/issues/3097

CUDA和Pytorch

如果你的程序成功运行却发现gpu占用率为0，那么说明CUDA没有正确配置与启用。要配置cuda，首先要知道cuda与pytorch版本的对应关系 https://pytorch.org/get-started/previous-versions/，然后按照网站上的命令安装对应的cuda。

强烈建议先通过这个程序进行检查

如果是False那就说明cuda版本不匹配，一定要下载安装匹配的版本

也可以参考https://matpool.com/supports/doc-public-data/#cuda-%E5%AE%89%E8%A3%85，但是官网提供的cuda版本有限，一定要和pytorch做好对应！
